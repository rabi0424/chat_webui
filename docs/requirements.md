# 要件定義書 — 個人用 AI チャット Web UI

- 版: v1.0（初版）
- 日付: 2026-07-05
- 状態: ドラフト（レビュー待ち）

---

## 1. プロジェクト概要

### 1.1 目的

ChatGPT / Claude のような AI チャットインターフェースを個人用に構築する。
OpenRouter API を通じて複数プロバイダのモデルを単一の UI から利用できるようにする（Poe に近い体験）。将来的には fal API 等にも対応する。

### 1.2 利用者・利用環境

| 項目 | 内容 |
|---|---|
| 利用者 | 本人のみ（シングルユーザー） |
| 端末 | iPhone（Safari）、Mac（Safari / Chrome） |
| アクセス形態 | Web ブラウザからいつでもアクセス（常時稼働のクラウドホスティング） |
| リポジトリ | GitHub 公開リポジトリ（完成後プライベート化） |

### 1.3 予算

- 無償が望ましい。上限は月額 500 円程度。
- 想定構成（後述）では通常利用で **月額 0 円** に収まる見込み。

## 2. システム構成（インフラ）

Cloudflare のサービス一式で構成する。個人利用の規模であれば全て無料枠内。

| 役割 | サービス | 備考 |
|---|---|---|
| ホスティング / API | Cloudflare Workers（+ 静的アセット） | SSR・API・SSE ストリーミング中継 |
| データベース | Cloudflare D1（SQLite） | 会話・メッセージ・ボット定義など |
| ファイルストレージ | Cloudflare R2 | 送受信画像・添付ファイル。egress 無料 |
| 認証 | Cloudflare Access（Zero Trust） | アプリ手前でメール OTP / Google IdP 認証 |
| デプロイ | GitHub 連携（Workers Builds） | push で自動デプロイ |

### 2.1 技術スタック

- **フロントエンド**: React。フレームワークは Cloudflare Workers 上でネイティブに動作する **React Router v7（旧 Remix）** を第一候補とする（Next.js は OpenNext 経由となりビルドが重いため次点）。
- **言語**: TypeScript
- **DB アクセス**: Drizzle ORM（D1 公式対応）
- **LLM ストリーミング**: OpenRouter の SSE を Workers 経由でクライアントへ中継

### 2.2 認証（Cloudflare Access）

- アプリの手前で Cloudflare Access が認証を行い、本人のメールアドレスのみ許可する。
- アプリ側の認証実装はほぼ不要。ただし多層防御として、Worker 側でも Access の JWT（`Cf-Access-Jwt-Assertion`）を検証する。
- セッション期間は長め（例: 1 ヶ月）に設定し、スマホでの再ログイン頻度を下げる。
- 注意: Access ポリシーを適用するには、ドメインが Cloudflare ゾーン上にあること（またはワンクリック Access 対応のあるデプロイ形態）が必要。無料で済まない場合の代替はカスタムドメイン取得（年 1,000〜1,500 円程度 → 月換算で予算内）。

### 2.3 API キーの扱い（セキュリティ要件）

- OpenRouter の API キーは **Workers の Secret としてサーバー側のみに保持** し、クライアントには一切渡さない。
- LLM への全リクエストは Worker を経由する（クライアント → Worker → OpenRouter）。
- 公開リポジトリであるため、キー・個人情報は一切コミットしない（`.dev.vars` / Secrets で管理）。

## 3. 機能要件

### 3.1 モデル管理【特別要件①】

- **OpenRouter のモデル一覧 API（`GET /api/v1/models`）を実行時に取得**し、アプリの実装を変更せずに常に最新のモデル一覧を閲覧・選択できる。
  - 一覧はサーバー側で短時間（例: 1 時間）キャッシュする。
- モデル一覧では以下ができる:
  - 名前・プロバイダによる検索 / フィルタ
  - コンテキスト長、入出力価格、モダリティ（画像入力可否）等のメタ情報表示
  - お気に入り登録（よく使うモデルを上位表示）
- 会話の途中でもモデルを切り替えられる。

### 3.2 ボット（プリセット）機能【特別要件②】

- 「モデル + システムプロンプト」の組み合わせを **ボット** として登録できる。
- ボットに含める項目:
  - 名前、アイコン（絵文字またはアップロード画像）
  - モデル ID
  - システムプロンプト
  - 生成パラメータ（temperature、max_tokens、top_p など。未指定はデフォルト継承）
- ボットの作成 / 編集 / 複製 / 削除ができる。
- 新規チャット開始時にボットを選択できる（ボットを使わない素のモデル選択も可能）。
- ボット編集は既存会話に遡及しない（会話開始時点の設定で継続。切替は明示操作）。

### 3.3 チャット（基本機能）

- 選択したモデル / ボットとのチャット。
- **ストリーミング表示**（SSE）。生成の停止ボタンあり。
- Markdown レンダリング（コードブロックのシンタックスハイライト、コピー機能、表、数式は将来対応可）。
- メッセージ操作:
  - 自分のメッセージの **編集して再送信**
  - アシスタント応答の **再生成**
  - 個別メッセージのコピー / 削除
- **画像添付**（マルチモーダル対応モデルへの画像入力）。画像は R2 に保存。
- エラー処理: API エラー・レート制限・残高不足を明示し、再試行できる。
- 使用量の目安表示: 応答ごとのトークン数・概算コスト（OpenRouter のレスポンスメタデータを利用）。

### 3.4 会話ブランチ【特別要件③】

- メッセージは **ツリー構造** で保持する（各メッセージが親メッセージ ID を持つ）。
- 任意の過去メッセージから **分岐（ブランチ）を作成** して別の続きを試せる。
- 「メッセージの編集して再送信」「応答の再生成」も内部的にはブランチとして保持し、破壊的に上書きしない。
- 兄弟ブランチ間の切り替え UI（例: `< 2/3 >` のようなページャ）を各分岐点に表示。
- 会話ごとに「現在表示中のリーフ（パス）」を記憶する。

### 3.5 会話管理

- 会話の一覧（サイドバー）、新規作成、削除、リネーム。
- タイトルの自動生成（最初のやり取りから安価なモデルで生成）。
- 会話のピン留め。
- 会話内容の全文検索（初期はタイトル + 簡易検索で可、全文検索は拡張）。
- すべての会話・添付ファイルはクラウド（D1 / R2）に保存され、端末をまたいで同期される。

### 3.6 設定画面

- デフォルトモデル / デフォルト生成パラメータの設定。
- デフォルトシステムプロンプトの設定。
- OpenRouter キーの残高表示（OpenRouter の credits API を利用、可能な範囲で）。
- 表示設定（テーマ: ライト / ダーク / システム追従）。

### 3.7 モバイル対応

- レスポンシブデザイン（スマホ幅ではサイドバーをドロワー化）。
- PWA 対応（ホーム画面追加、アプリ風の全画面表示）。オフライン動作は要件外。

## 4. 非機能要件

| 項目 | 要件 |
|---|---|
| コスト | 通常利用で月額 0 円（Cloudflare 無料枠内）。上限 500 円/月 |
| セキュリティ | Cloudflare Access による本人限定アクセス。API キーはサーバー側 Secret のみ。Worker 側でも Access JWT を検証 |
| 可用性 | Cloudflare 依存で十分（個人利用のため SLA 要件なし） |
| 性能 | 初回表示が軽量であること。ストリーミングは体感遅延なく表示 |
| データ保全 | D1 の定期エクスポート手段を用意（手動コマンドで可） |
| 保守性 | プロバイダ（OpenRouter / fal / …）を抽象化した内部インターフェースを設け、追加プロバイダ対応時に UI 層の変更を最小化する |

## 5. データモデル（案）

```
bots
  id, name, icon, model_id, system_prompt,
  params_json, created_at, updated_at

conversations
  id, title, bot_id (nullable), model_id,
  pinned, current_leaf_message_id,
  created_at, updated_at

messages                      -- ツリー構造
  id, conversation_id,
  parent_id (nullable),       -- NULL はルート
  role ('user' | 'assistant' | 'system'),
  content,
  model_id,                   -- 応答生成に使ったモデル
  usage_json,                 -- トークン数・コスト
  created_at

attachments
  id, message_id, r2_key, mime_type, size, created_at

settings
  key, value_json              -- デフォルトモデル等
```

- ブランチは `messages.parent_id` によるツリーで表現。兄弟ノード = 分岐。
- `conversations.current_leaf_message_id` で表示中パスを復元。

## 7. 開発フェーズ

### Phase 1 — MVP
- Cloudflare 環境構築（Workers + D1 + R2 + Access + GitHub 自動デプロイ）
- モデル一覧の動的取得・選択【要件①】
- ストリーミングチャット（Markdown 表示、停止、再生成）
- 会話の保存・一覧・削除・タイトル自動生成

### Phase 2 — 特別要件の完成
- ボット登録機能【要件②】
- 会話ブランチ【要件③】（ツリー保存は Phase 1 のデータモデルから対応）
- 画像添付（マルチモーダル入力、R2 保存）

### Phase 3 — 快適化
- PWA 対応、モバイル UI 磨き込み
- 検索、ピン留め、使用量 / 残高表示
- 設定画面の拡充

### Phase 4 — 拡張
- fal API 対応（画像生成）
- その他プロバイダ追加
